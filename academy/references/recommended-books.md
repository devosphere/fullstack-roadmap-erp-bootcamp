# ERP Bootcamp Recommended Books

**Document Version:** 1.0  
**Purpose:** Provide a curated reading list to develop professional software engineering, architecture, business analysis, and leadership skills.

---

# Introduction

Books provide deeper understanding beyond tutorials and documentation.

The ERP Bootcamp recommends books that develop:

- Engineering fundamentals
- Software craftsmanship
- System design thinking
- Business understanding
- Architecture skills
- Product mindset
- Leadership capability

The objective is not to read everything at once.

The learner should read according to the current Sprint phase.

---

# Reading Philosophy

Recommended approach:

```
Read Concept

↓

Apply To ERP Project

↓

Document Learning

↓

Discuss Design Decisions

↓

Improve Implementation
```

A book is valuable only when knowledge is applied.

# 1. Software Engineering Fundamentals

---

# The Pragmatic Programmer

**Authors:** Andrew Hunt & David Thomas

## Recommended Phase

Sprint 00+

## Why Read It

Develops the mindset of a professional software engineer.

Topics:

- Problem solving
- Code ownership
- Software craftsmanship
- Continuous learning
- Engineering discipline

## Key Lessons

A good engineer:

- Takes responsibility for code quality.
- Automates repetitive work.
- Thinks about maintainability.
- Improves continuously.

---

# Clean Code

**Author:** Robert C. Martin

## Recommended Phase

Sprint 01+

## Why Read It

Teaches how to write maintainable software.

Topics:

- Naming
- Functions
- Classes
- Refactoring
- Code organization

## Key Lessons

Code is read more often than it is written.

Readable code reduces:

- Bugs
- Maintenance cost
- Team friction

# Final Principle

A great engineer combines:

```
Programming Skills

+

System Design

+

Business Understanding

+

Quality Mindset

+

Communication

+

Continuous Learning
```

The purpose of this reading list is not to create someone who knows many technologies.

The purpose is to create an engineer capable of designing, building, and operating complete business systems.
